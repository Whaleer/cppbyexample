### ✅ 设计原则说明
1. **原子性**：每个 example 只演示 1 个核心概念（如“引用”不混入“指针”）
2. **依赖最小化**：example N 不依赖 example N+1 的知识
3. **认知平滑**：难点拆解（如指针拆为 24/25/26 三个 example）
4. **对比教学**：关键对比单独成例（如 28 指针 vs 引用）
5. **现代优先**：`nullptr` 早于 `NULL`，`std::string` 早于 C 风格字符串
